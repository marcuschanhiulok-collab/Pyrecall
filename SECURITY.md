# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.10.x  | :white_check_mark: |
| < 0.10  | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Pyrecall, please open a public GitHub issue.

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested fixes, if you have them

You can expect an acknowledgement within **48 hours** and a status update within **7 days**. If the vulnerability is accepted, a patch will be released as soon as possible and you will be credited in the release notes. If declined, we will explain why.
